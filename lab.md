Варіант3: Трекер води — додавання випитих склянок води протягом дня.
Мета продукту: Користувач повинен мати можливість **легко і швидко реєструвати кількість випитої води протягом дня, щоб відстежувати свій прогрес у досягненні щоденної норми**
**Аналіз вимог**
1. Як користувач, я хочу мати можливість додавати випиту склянку води одним натисканням, щоб максимально швидко реєструвати споживання 
2. Як користувач, я хочу бачити загальну кількість випитої води та прогрес до моєї цілі, щоб розуміти, наскільки я близько до виконання норми
3. Як користувач, я хочу встановлювати свою щоденну ціль по воді (у літрах чи склянках), щоб трекер був персоналізований
4. Як користувач, я хочу мати можливість відредагувати або видалити запис про воду, щоб виправити помилки в обліку
5. Як користувач, я хочу отримувати нагадування, якщо я довго не пив воду, щоб підтримувати регулярну гідратацію

Великий графік або індикатор, що показує: "Випито 1.2 л / Ціль 2.0 л"
Велика кнопка "+" або іконка склянки для швидкого додавання порції (наприклад, 250 мл)
Кнопка налаштувань
**Реалізація (імітація)**
Ключова функція: Реєстрація випитої склянки води
**Псевдокод**
// Змінна для зберігання загального об'єму, випитого за день
ЗМІННА ЗагальнийОб'ємСьогодні = 0 
ЗМІННА Об'ємПорціїЗаЗамовчуванням = 250 // мл

ФУНКЦІЯ ДодатиПорціюВоди(Порція = Об'ємПорціїЗаЗамовчуванням):
    // 1. Перевірка коректності вхідних даних
    ЯКЩО Порція <= 0:
        ПОВІДОМИТИ КОРИСТУВАЧА "Об'єм повинен бути більше нуля."
        ПОВЕРНУТИ ПОМИЛКА
    КІНЕЦЬ ЯКЩО

    // 2. Оновлення загального об'єму
    ЗагальнийОб'ємСьогодні = ЗагальнийОб'ємСьогодні + Порція

    // 3. Збереження запису в історії
    СтворитиЗапис({
        Час: ПоточнийЧас(),
        Об'єм: Порція
    })
    
    // 4. Оновлення інтерфейсу
    ОновитиГрафікПрогресу(ЗагальнийОб'ємСьогодні)
    
    ПОВЕРНУТИ УСПІХ
КІНЕЦЬ ФУНКЦІЇ
**Тестування** 
1. Успішне додавання: Натиснути кнопку "+" (додається 250 мл) ---- Загальний об'єм збільшується на 250 мл, графік прогресу оновлюється
2. Негативне значення (помилка): Спроба додати об'єм -100 мл (через спеціальний інтерфейс/введення) ---- Функція повертає "ПОМИЛКА", загальний об'єм не змінюється, з'являється повідомлення користувачу
3. Перевірка цілі: Загальний об'єм досягає або перевищує встановлену ціль (наприклад, 2000 мл) ----- З'являється повідомлення-привітання "Ціль досягнута!", індикатор прогресу відображає 100%+
**Висновок**
Для розробки простого застосунку "Трекер води" найкраще підходить Agile-підхід (Гнучка методологія). Він дозволить швидко реагувати на відгуки користувачів щодо зручності додавання води та ітераційно додавати складніші функції (нагадування, історія, графіки) кожні 2–4 тижні. Waterfall був би занадто жорстким, а Spiral — надмірно складним для такого мінімального життєздатного продукту (MVP)
